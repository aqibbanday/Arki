# Arki Design Laws

The Design Laws are non-negotiable rules that every component, pattern and application built with Arki must follow.

## Law 1 — One Primary Action
Every view must present a clear primary action.

## Law 2 — Motion Has Meaning
Animation must communicate state, hierarchy or continuity.

## Law 3 — Whitespace Is Functional
Whitespace improves comprehension and visual rhythm.

## Law 4 — Systems Before Solutions
Solve problems at the framework level whenever possible.

## Law 5 — Performance Is Visible
A fast interface increases user trust.

## Law 6 — Design for Change
Components should evolve without breaking consumers.

## Law 7 — APIs Should Read Naturally
Public APIs should be expressive, discoverable and consistent.

## Law 8 — Premium Through Restraint
Avoid unnecessary decoration. Let clarity define quality.
