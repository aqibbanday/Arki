# Core Principles

These principles govern every design, engineering and product decision within Arki.

## 1. Design with Intent
Every element must exist for a reason.

## 2. System over Screens
Build reusable systems instead of page-specific solutions.

## 3. Performance by Default
Performance is part of the initial design, not a later optimization.

## 4. Semantic Everything
Use semantic design tokens, component names and APIs.

## 5. Accessibility Always
Keyboard navigation, screen readers, contrast and scaling are mandatory.

## 6. Consistency Builds Trust
Users should learn patterns once and reuse that knowledge everywhere.

## 7. Composition over Complexity
Compose small reusable components instead of large specialized controls.

## 8. Documentation is a Feature
No public component is complete without a specification.

## 9. Developer Experience Matters
The API should be discoverable, predictable and enjoyable.

## 10. Long-term Thinking
Every decision should still make sense years from now.
